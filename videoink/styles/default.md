# default style

Neutral, informative, section-headed. Suitable for general blog posts,
newsletters, and knowledge articles.

## Rules

- Preserve the speaker's original argument structure; do not invent new
  sections or claims.
- Remove verbal tics, repetition, filler, and sponsor / intro / outro
  segments.
- Use descriptive H2 headings, one topic per section.
- Keep paragraphs short (2-4 sentences).
- Cite specific facts, numbers, names, dates from the transcript; do not
  hallucinate unknown details.
- Do not use meta phrases like "In this article we will...", "As
  mentioned above...", "In conclusion...".
- If the transcript does not support a claim, omit it rather than
  speculating.
- Output plain Markdown only; no HTML, no YAML frontmatter.

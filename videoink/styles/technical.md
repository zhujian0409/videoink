# technical style

For engineering blog posts and developer-audience articles.

## Rules

- Lead with the problem the speaker is solving, not the tool.
- Preserve code snippets, commands, error messages, and output shown in
  the video verbatim; use fenced code blocks with a language hint.
- Use present tense and active voice. Use "you" where the speaker does.
- Prefer links to primary sources (official docs, RFCs, source code)
  over secondary blog references.
- Call out tradeoffs, prerequisites, and version-specific quirks when
  the speaker mentions them.
- Use bulleted lists for enumerations rather than long comma-separated
  lines.
- Inline `fixed-width` for file paths, identifiers, commands, and flags.
- Omit marketing language ("powerful", "revolutionary", "cutting-edge").
